> **Opt-in skill.** Per the Sprint Spine invocation discipline: invoke explicitly for a high-leverage strategic call. Lead with the recommendation — hedging in advance is not analysis.

## When to use

When the operator wants the sharp founder/CEO take on something already on the table — "go/no-go", "what's your take", "expand or cut", "prioritize this". A strategic gut-check, not the full planning pipeline.

- **ceo-review** → sharp call on a decision / feature / tradeoff already scoped.
- **office-hours** → instead, for a raw, unvalidated idea ("should this exist?").
- **sprint-plan** → instead, when planning the build of an approved deliverable.

The orchestrator dispatches one CEO/founder specialist; it does not write the call itself.

## Process

Do not validate weak framing. Work the lens, briefly and concretely:

1. **The decision** — what decision are we actually making? Strip it to the real fork.
2. **10-star reframe** — what's the 10-star version, not the safe one in the request? Expand scope where it creates a genuinely better product; cut what isn't load-bearing. The great version is often bigger, not smaller — but bloat is not ambition.
3. **Killer assumption** — the single belief that must be true. State it falsifiably.
4. **Failure mode** — what most likely breaks the plan.
5. **Evidence** — split confirmed / assumed / unknown. Never let an assumption pass as fact.
6. **Prioritization** — now / later / never, against the alternative use of the same time. Strategy is not execution: a good build can still be the wrong priority.
7. **Next move** — the smallest action that creates real signal.

## Output

A tight founder call:
- **Call** — GO / NO-GO / WAIT / NEEDS DECISION
- **Recommendation** — one clear paragraph (pick a lane; no hedging)
- **10-star reframe** · **Rationale** (3 bullets) · **Killer assumption + top risk** · **Cut / keep / sequence** · **Next move** (action + the signal that proves it worked)

Direct, decisive, anti-sycophantic.

## Gate

Standalone advisory gate — no artifact, no pipeline. For public, legal, financial, or otherwise irreversible decisions, end at an explicit operator approval gate — recommend, do not trigger. If the call is "build it," the next step is `office-hours` (if still unvalidated) or `sprint-think` (if approved) — never auto-chained.
